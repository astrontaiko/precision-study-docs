机器备课系统逻辑说明书

一、概述

本系统采用“内容结构化 + 学生分层 + 策略规则化”的方式，实现面向一类学生的个性化学习方案推荐。核心思想是通过对教学内容进行精细化的原子拆解与标签标注，结合学生的学习数据和校内进度，自动匹配适合的学习策略与资源组合。

---

二、内容层结构

2.1 课程地图

由“学科 + 年级 + 版本 + 册别”唯一确定一个课程地图，例如“小学数学人教版一年级下册”。

2.2 四层内容结构

层级 名称 说明

L1 章 教材的自然章节划分

L2 节 章节下的节

L3 元

L4 题模 承载具体教学内容

2.3 学习方案

每个题模下可配置多种学习方案，主要类型包括：

· 新知学方案：通常用于初次学习新知识点

· 巩固学方案：通常用于复习、强化训练

每个方案内包含具体的学习资源，如视频、交互学件、题目等。

---

三、题模标签体系

为实现精细化筛选，每个题模配置以下三个维度的标签：

3.1 难度标签

级别 说明

基础 核心概念、基础题型

进阶 变式题、简单综合

拔高 中等难度综合题

超越 高难度拓展题

3.2 重要度标签

级别 说明

高 核心考点、必学内容

中 重要但非核心

低 辅助性内容

3.3 知识类型标签

类型 说明

认知增量 必须先通过讲解（如视频）才能学会的知识点

融会贯通 通过做题即可掌握的题型类知识，依赖前置知识

---

四、学生分层策略

系统按学生成绩将学生划分为若干层级（如90分以上、70-90分、70分以下等），每个层级的学生共用一套学习策略，而非完全个性化的千人千面。

---

五、学习场景

场景 说明

超前预习 学习尚未在校内讲授的内容，侧重基础概念引入

同步校内 匹配当前校内教学进度，侧重巩固与查漏补缺

两个场景的策略框架一致，仅内容侧重不同。

---

六、策略匹配核心逻辑

6.1 第一步：题模筛选

根据学生所属的分层和场景，通过题模标签组合筛选目标题模集：

筛选条件 = 难度（如基础）+ 重要度（如高）+ 知识类型（如认知增量）

三个维度同时满足，选出匹配的题模集合。

6.2 第二步：通关方式判断

系统根据是否有学习数据，进入不同分支：

```

筛选题模

↓

是否有系统学习数据？

├── 有 → 按正确率分流

└── 无 → 按校内进度分流

```

6.2.1 有学习数据分支

根据该题模的历史正确率，决定通关方式：

正确率条件 通关方式 说明

< 90%（或阈值） 测学 先测一道题，测对则通过，测错则进入学习

≥ 90%（或阈值） 学 直接进入学习（或跳过）

注：具体阈值可根据分层和场景灵活配置

6.2.2 无学习数据分支

通过校内进度判断学生是否接触过该知识点：

校内进度 说明 通关方式

未学到 该节在当前校内进度之后，属全新知识 学（新知学方案）

已学过 该节在当前校内进度之前，已有基础 测学

校内进度的确定：系统在创建学习规划时，由用户选择当前校内进度（具体到某一节），该节之前视为“已学过”，该节及之后视为“未学到”。

6.3 通关方式说明

方式 说明

学 直接学习该题模下的完整学习方案（如新知学方案）

测学 先进行一道检测题，若答对则跳过该题模，若答错则进入学习方案

---

七、方案与资源

学习方案由专业学科老师预先设计，包括：

· 新知学方案

· 巩固学方案

每个方案内可包含多种资源类型：视频、交互学件、题目等。

系统在决策时，根据场景（预习/同步）和通关方式（学/测学）自动选择对应方案。

九、设计特点总结

1. 内容结构化：将教学内容拆解到题模层级，确保可复用、可组合

2. 标签驱动：通过难度、重要度、知识类型三个维度实现内容的精细筛选

3. 分层策略：面向一类学生而非个体，平衡个性化与规模化

4. 双分支决策：区分有/无学习数据，分别用正确率和校内进度作为判断依据

5. 测学过机制：通过“测学”减少无效学习，提升学习效率

6. 方案预制：学习方案由专业老师设计，保证内容质量
