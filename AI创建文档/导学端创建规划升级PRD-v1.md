# 导学端创建规划升级 PRD v1

## 1. 文档信息

- 产品名称：精准学导学端
- 需求名称：创建规划升级 + 地图页策略摘要
- 版本号：V1.0
- 更新时间：2026-03-26
- 文档目的：明确导学端在“创建规划”和“地图查看规划”两个关键环节的产品需求，支撑原型、评审与研发实现。

## 2. 需求背景

当前导学端在创建学生学习规划时，存在以下问题：

1. 导学师需要自己判断学生适合什么内容、从哪里开始学、当前更适合预习还是提升，依赖个人经验较重。
2. 学生标签建立方式不够标准，当前主要依赖老师主观判断，缺少结构化输入。
3. 地图页只展示学习内容结果，缺少对当前规划逻辑的解释信息，导学师无法快速理解为什么这样规划。
4. 新手导学师面对较多地图信息和调整选项时，容易不知道该如何操作。

因此，本期需要围绕两个方向进行升级：

- 升级创建规划流程：以“直接选择学生标签”为主，并提供“辅助判断”入口帮助老师确认能力和意愿
- 升级地图页信息展示：增加策略摘要，让导学师更容易理解当前规划结果

## 3. 产品目标

### 3.1 业务目标

- 降低导学师创建规划的门槛和耗时
- 提升规划标准化程度
- 提升导学师对规划结果的理解和执行信心

### 3.2 产品目标

- 支持两种创建规划方式：
  - 直接选择学生标签
  - 通过问题辅助判断标签
- 在地图页展示策略摘要
- 让导学师能快速理解“这个学生为什么这样学”

## 4. 用户角色

- 直接用户：导学师
- 间接受益用户：学生、家长、学科团队、运营管理者

## 5. 核心场景

### 5.1 首次创建规划

导学师首次为学生创建规划，需要快速完成学生画像确认并生成初始规划。

### 5.2 经验不足导学师创建规划

导学师不确定学生能力或学习意愿时，可通过系统问题辅助判断标签，降低人工决策门槛。

### 5.3 已创建规划后回看地图

导学师需要在地图页快速了解：

- 当前学生属于什么类型
- 当前规划覆盖范围是什么
- 当前策略为什么这样设计

## 6. 功能范围

本期功能仅聚焦导学端：

- 创建规划弹窗升级
- 地图页增加策略摘要

不在本期展开：

- 课后作业自动生成逻辑
- 后台策略平台配置逻辑
- 试卷分析与AI规划逻辑

## 7. 页面范围

### 7.1 创建规划弹窗

入口：地图页右上角“+规划”按钮

### 7.2 地图页右侧摘要区

新增策略摘要模块

## 8. 详细需求

## 8.1 创建规划弹窗

### 8.1.1 页面目标

让导学师以低负担方式完成学生标签确认，并基于标签创建学习规划。

### 8.1.2 创建方式

创建规划采用“直接选标签”为主流程。

老师进入弹窗后，直接选择以下核心信息：

- 学习成绩
- 学习能力
- 学习意愿
- 学习阶段
- 校内进度（仅同步提升时需要）

对于不确定学生能力和意愿的老师，页面右上角提供“辅助判断”入口。点击后打开独立小弹窗，通过若干选择题辅助判断学生能力和意愿，并在点击“确定”后，把结果同步回主表单中的能力和意愿标签。

### 8.1.3 字段设计

#### 1. 学习成绩

- 类型：单选
- 选项：
  - 95分以上
  - 70-94分
  - 70分以下
- 必填：是

#### 2. 学习能力

- 直接模式：直接单选高/中/低
- 辅助判断入口：通过问题弹窗自动判断后回填
- 必填：是

#### 3. 学习意愿

- 直接模式：直接单选高/中/低
- 辅助判断入口：通过问题弹窗自动判断后回填
- 必填：是

#### 4. 学习阶段

- 类型：单选或下拉
- 选项：
  - 超前预习
  - 同步提升
- 必填：是

#### 5. 校内进度

- 类型：下拉
- 必填条件：
  - 当学习阶段为“同步提升”时必填
  - 当学习阶段为“超前预习”时不展示、不填写
- 说明：用于确定当前校内教学进度，帮助系统判断哪些内容属于校内已学

#### 6. 备注

- 类型：文本
- 必填：否
- 长度限制：50 字

### 8.1.4 辅助判断弹窗

辅助判断为独立弹窗，不直接替代主表单。

页面结构：
- 学习能力判断题
- 学习意愿判断题
- 判断结果展示
- “确定”按钮

点击“确定”后：
- 将能力结果同步回主表单中的“学习能力”
- 将意愿结果同步回主表单中的“学习意愿”
- 关闭辅助判断弹窗

### 8.1.5 能力判断题示例

- 能自己总结知识重点，而不是机械记忆
- 学过的内容能迁移到新题目里
- 做题时能较快定位思路
- 知识讲过一次后基本能理解并应用

### 8.1.6 意愿判断题示例

- 会主动完成学习任务，不太需要催促
- 遇到不会的题愿意继续尝试
- 对提分或新知识有明显兴趣
- 在课堂中愿意参与互动

### 8.1.7 交互要求

- 主流程默认直接选标签
- 页面右上角提供“辅助判断”按钮
- 辅助判断弹窗中完成作答后，点击“确定”才同步标签结果
- 选择“同步提升”时展示“校内进度”
- 选择“超前预习”时隐藏“校内进度”
- 右侧规划预览在核心信息填写完整前不展示
- 核心信息填写完整后，右侧规划预览出现
- 点击“新建规划”后，关闭弹窗并更新地图页摘要

## 8.2 地图页策略摘要

### 8.2.1 目标

让导学师理解当前规划背后的核心策略逻辑。

### 8.2.2 展示内容

地图页右侧不再展示“规划概况”，统一展示策略摘要模块。

本期先按两类策略摘要设计：

- 学习策略
- 作业策略

#### 学习策略示例：

- 优先学习重点基础内容，确保同步校内节奏
- 已学且掌握好的简单内容减少重复学习
- 已学但掌握不好的内容进入测学与巩固

#### 作业策略示例：

- 课后自动生成日常作业
- 作业优先包含近期错题
- 章节学完后自动安排章节复习任务

### 8.2.3 展示要求

- 摘要应为自然语言，不能只展示底层规则字段
- 信息需让导学师“看得懂、能解释”
- 不展示过多技术细节，强调策略结论

## 9. 标签与学生类型规则

### 9.1 成绩档位

- ≥95：优秀
- 70~94：中等
- ≤69：后进

### 9.2 能力档位

- 高
- 中
- 低

### 9.3 意愿档位

- 高
- 中
- 低

### 9.4 学生类型示例映射

- 成绩好 + 能力高 + 意愿高：优等生/优潜生
- 成绩中等 + 能力弱 + 意愿强：努力的中等生
- 成绩中等 + 能力强 + 意愿弱：意愿不足的中等生
- 成绩中等 + 能力弱 + 意愿弱：薄弱的中等生
- 成绩较低 + 能力弱 + 意愿弱：后进生

## 10. 状态设计

### 10.1 创建规划状态

- 未规划
- 规划已生成
- 规划已更新

### 10.2 模式状态

- 直接选标签模式
- 问题辅助判断模式

## 11. 校验规则

- 学习成绩不能为空
- 能力和意愿不能为空
- 学习阶段不能为空
- 当学习阶段为“同步提升”时，校内进度不能为空

## 12. 埋点建议

- 点击“+规划”
- 点击“辅助判断”
- 修改学习目标
- 完成能力判断
- 完成意愿判断
- 点击“新建规划”
- 查看策略摘要
